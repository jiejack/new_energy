# 错误处理流程

## 后端错误处理

### 1. 错误定义
- 使用自定义错误类型，如 `ErrConfigNotFound`、`ErrInvalidValueType` 等
- 错误信息应清晰明确，便于前端理解

### 2. 错误传递
- 从 Repository 层到 Service 层再到 Handler 层，错误应逐级传递
- Service 层可以对原始错误进行包装，添加更多上下文信息

### 3. 错误响应
- Handler 层负责将错误转换为 HTTP 响应
- 使用统一的错误响应格式：
  ```json
  {
    "code": 400,
    "message": "错误信息",
    "timestamp": 1620000000
  }
  ```

### 4. 错误状态码
- 400: 请求参数错误
- 404: 资源不存在
- 500: 服务器内部错误

## 前端错误处理

### 1. API 错误处理
- 使用 axios 拦截器统一处理 API 错误
- 对不同错误状态码进行不同处理

### 2. 表单验证错误
- 使用 Element Plus 的表单验证
- 实时显示验证错误信息

### 3. 全局错误处理
- 实现全局错误处理机制
- 对于非预期错误，显示友好的错误提示

### 4. 错误日志
- 前端错误应记录到控制台
- 关键错误应上报到后端
