## 🔍 深度分析

### 推理服务需求分析

#### 1. 实时推理需求
- **场景**: 实时预测、异常检测、即时预警
- **响应时间**: < 100ms (P95)
- **并发量**: 支持1000+ QPS
- **应用**: 实时数据点预测、故障即时识别

#### 2. 批量推理需求
- **场景**: 批量预测、历史数据重算、离线分析
- **处理能力**: 支持100万+数据点/小时
- **处理时间**: 灵活调度，非实时性要求
- **应用**: 日报/周报生成、历史趋势分析

#### 3. 混合推理需求
- **场景**: 预测模型需要同时支持实时和批量
- **资源隔离**: 实时和批量任务资源隔离
- **优先级控制**: 实时任务优先级高于批量

#### gRPC API 设计（高性能场景）

```protobuf
syntax = "proto3";

package ai.inference;

service InferenceService {
  rpc Predict(PredictRequest) returns (PredictResponse);
  rpc PredictStream(stream PredictRequest) returns (stream PredictResponse);
  rpc BatchPredict(BatchPredictRequest) returns (BatchPredictResponse);
}

message PredictRequest {
  string model_id = 1;
  string version = 2;
  map<string, Value> inputs = 3;
  PredictOptions options = 4;
}

message PredictResponse {
  string request_id = 1;
  string model_id = 2;
  string version = 3;
  Prediction prediction = 4;
  Explanation explanation = 5;
  InferenceMetadata metadata = 6;
}

message PredictOptions {
  bool include_confidence = 1;
  bool include_explanation = 2;
  int32 cache_ttl_seconds = 3;
}
```

## 💾 推理缓存策略设计

### 缓存层级架构

```
┌─────────────────────────────────────────┐
│         L1: 本地内存缓存                 │
│    (超低延迟，容量有限，热点数据)        │
└───────────────┬─────────────────────────┘
                │ 未命中
                ▼
┌─────────────────────────────────────────┐
│         L2: Redis 分布式缓存             │
│    (低延迟，大容量，共享缓存)            │
└───────────────┬─────────────────────────┘
                │ 未命中
                ▼
┌─────────────────────────────────────────┐
│         L3: 模型推理计算                 │
│    (正常延迟，计算开销大)                │
└─────────────────────────────────────────┘
```

### 缓存键设计

```
格式: ai:predict:{model_id}:{version}:{feature_hash}

示例:
ai:predict:solar_forecast_v1.0:latest:abc123def456

feature_hash = SHA256(sorted_input_features_json)
```

### 缓存失效策略

| 策略类型 | 触发条件 | 适用场景 |
|---------|---------|---------|
| TTL失效 | 时间到期 | 所有场景默认 |
| 版本失效 | 模型版本更新 | 模型升级时 |
| 主动失效 | 数据重大变化 | 训练数据更新 |
| LRU淘汰 | 缓存容量满 | 热点数据更新 |

### 缓存预热机制

```
预热策略:
1. 定时预热: 每天凌晨预测下一天的热点数据
2. 智能预热: 基于历史访问模式预热高频查询
3. 增量预热: 新模型部署后自动预热热点数据

预热范围:
- Top 100 高频查询
- 关键场站预测
- 近期历史查询
```

## 📝 数据库Schema设计（推理相关）

### 预测结果表

```sql
CREATE TABLE prediction_results (
    id BIGSERIAL PRIMARY KEY,
    request_id VARCHAR(64) NOT NULL,
    model_id VARCHAR(128) NOT NULL,
    model_version VARCHAR(32) NOT NULL,
    station_id VARCHAR(64) NOT NULL,
    device_id VARCHAR(64),
    prediction_time TIMESTAMP NOT NULL,
    target_time TIMESTAMP NOT NULL,
    prediction_value FLOAT NOT NULL,
    prediction_unit VARCHAR(32),
    confidence FLOAT,
    confidence_lower FLOAT,
    confidence_upper FLOAT,
    actual_value FLOAT,
    error FLOAT,
    error_percentage FLOAT,
    input_features JSONB,
    metadata JSONB,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    INDEX idx_prediction_model (model_id, model_version),
    INDEX idx_prediction_station (station_id, target_time),
    INDEX idx_prediction_time (target_time)
);
```

### 批量任务表

```sql
CREATE TABLE batch_inference_jobs (
    id BIGSERIAL PRIMARY KEY,
    job_id VARCHAR(64) NOT NULL UNIQUE,
    model_id VARCHAR(128) NOT NULL,
    model_version VARCHAR(32) NOT NULL,
    status VARCHAR(32) NOT NULL,
    total_items INTEGER NOT NULL,
    processed_items INTEGER DEFAULT 0,
    failed_items INTEGER DEFAULT 0,
    callback_url VARCHAR(512),
    result_url VARCHAR(512),
    error_message TEXT,
    started_at TIMESTAMP,
    completed_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    INDEX idx_job_status (status, created_at),
    INDEX idx_job_model (model_id, created_at)
);
```
