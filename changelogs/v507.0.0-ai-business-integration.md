## 💾 预测结果存储方案

### 数据库表结构设计

```sql
-- 预测结果主表
CREATE TABLE prediction_results (
    id BIGSERIAL PRIMARY KEY,
    prediction_id VARCHAR(64) NOT NULL UNIQUE,
    request_id VARCHAR(64) NOT NULL,
    model_id VARCHAR(128) NOT NULL,
    model_version VARCHAR(32) NOT NULL,
    station_id VARCHAR(64) NOT NULL,
    device_id VARCHAR(64),
    point_id VARCHAR(64),
    
    prediction_type VARCHAR(32) NOT NULL, 
    prediction_time TIMESTAMP NOT NULL,
    target_time TIMESTAMP NOT NULL,
    
    prediction_value FLOAT NOT NULL,
    prediction_unit VARCHAR(32),
    confidence FLOAT,
    confidence_lower FLOAT,
    confidence_upper FLOAT,
    
    actual_value FLOAT,
    error FLOAT,
    error_percentage FLOAT,
    accuracy_score FLOAT,
    
    input_features JSONB,
    explanation JSONB,
    metadata JSONB,
    
    status VARCHAR(32) DEFAULT 'active',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    INDEX idx_prediction_model (model_id, model_version),
    INDEX idx_prediction_station (station_id, target_time),
    INDEX idx_prediction_time (target_time),
    INDEX idx_prediction_type (prediction_type, created_at),
    INDEX idx_prediction_status (status, created_at)
);

-- 预测结果分区表（按时间分区）
CREATE TABLE prediction_results_partition OF prediction_results
PARTITION BY RANGE (target_time);

-- 批量预测任务表
CREATE TABLE batch_prediction_jobs (
    id BIGSERIAL PRIMARY KEY,
    job_id VARCHAR(64) NOT NULL UNIQUE,
    batch_id VARCHAR(64),
    model_id VARCHAR(128) NOT NULL,
    model_version VARCHAR(32) NOT NULL,
    
    station_id VARCHAR(64),
    device_id VARCHAR(64),
    
    status VARCHAR(32) NOT NULL,
    total_items INTEGER NOT NULL,
    processed_items INTEGER DEFAULT 0,
    failed_items INTEGER DEFAULT 0,
    
    callback_url VARCHAR(512),
    result_url VARCHAR(512),
    error_message TEXT,
    
    started_at TIMESTAMP,
    completed_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    INDEX idx_job_status (status, created_at),
    INDEX idx_job_model (model_id, created_at),
    INDEX idx_job_station (station_id, created_at)
);

-- 预测模型元数据表
CREATE TABLE prediction_models (
    id BIGSERIAL PRIMARY KEY,
    model_id VARCHAR(128) NOT NULL UNIQUE,
    version VARCHAR(32) NOT NULL,
    type VARCHAR(32) NOT NULL,
    name VARCHAR(256) NOT NULL,
    description TEXT,
    
    status VARCHAR(32) DEFAULT 'active',
    is_default BOOLEAN DEFAULT false,
    
    training_data_start TIMESTAMP,
    training_data_end TIMESTAMP,
    last_trained_at TIMESTAMP,
    last_deployed_at TIMESTAMP,
    
    metrics JSONB,
    hyperparameters JSONB,
    features JSONB,
    config JSONB,
    
    created_by VARCHAR(64),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    INDEX idx_model_type (type, status),
    INDEX idx_model_status (status, created_at)
);

-- 预测准确度历史表
CREATE TABLE prediction_accuracy_history (
    id BIGSERIAL PRIMARY KEY,
    model_id VARCHAR(128) NOT NULL,
    version VARCHAR(32) NOT NULL,
    station_id VARCHAR(64),
    
    date DATE NOT NULL,
    
    total_predictions INTEGER,
    accurate_predictions INTEGER,
    accuracy_rate FLOAT,
    
    mae FLOAT,
    rmse FLOAT,
    mape FLOAT,
    r_squared FLOAT,
    
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    UNIQUE(model_id, version, station_id, date),
    INDEX idx_accuracy_model (model_id, version, date),
    INDEX idx_accuracy_station (station_id, date)
);
```

### 存储策略

| 数据类型 | 存储位置 | 保留时间 | 压缩方式 |
|---------|---------|---------|---------|
| 实时预测结果 | 主表 + 热分区 | 30天 | LZ4 |
| 历史预测结果 | 冷分区 | 1年 | ZSTD |
| 归档预测结果 | 对象存储 | 5年+ | GZIP |
| 模型元数据 | 主表 | 永久 | 无 |
| 准确度历史 | 主表 | 2年 | LZ4 |

## ⚠️ 告警触发逻辑设计

### 告警触发流程

```
预测结果生成
    ↓
阈值检查
    ↓
趋势分析
    ↓
异常检测
    ↓
规则匹配
    ↓
告警生成
    ↓
级别判定
    ↓
通知发送
    ↓
状态跟踪
```

### 告警规则配置

```yaml
alarm_rules:
  - id: "solar_low_output"
    name: "光伏发电量过低告警"
    model_id: "solar_forecast_v1.0"
    condition:
      type: "threshold"
      field: "prediction_value"
      operator: "lt"
      value: 100
      unit: "kW"
    level: "warning"
    duration: "1h"
    enabled: true
  
  - id: "fault_high_risk"
    name: "设备高风险故障告警"
    model_id: "fault_detector_v1.0"
    condition:
      type: "threshold"
      field: "prediction_value"
      operator: "gt"
      value: 0.7
    level: "critical"
    duration: "5m"
    enabled: true
  
  - id: "prediction_deviation"
    name: "预测偏差过大告警"
    condition:
      type: "comparison"
      field: "error_percentage"
      operator: "gt"
      value: 20
      reference_field: "actual_value"
    level: "info"
    duration: "6h"
    enabled: true
```

### 告警级别定义

| 级别 | 说明 | 响应时间 | 通知方式 |
|-----|------|---------|---------|
| Critical | 紧急，需立即处理 | < 5分钟 | 电话、短信、APP推送 |
| Warning | 警告，需尽快处理 | < 30分钟 | 短信、APP推送、邮件 |
| Info | 信息，关注即可 | < 2小时 | 邮件、系统通知 |
| Low | 低优先级，定期查看 | < 24小时 | 系统通知 |

### WebSocket 实时推送

```javascript
// 连接WebSocket
const ws = new WebSocket('wss://api.example.com/ws/ai');

// 订阅实时预测
ws.send(JSON.stringify({
  type: 'subscribe',
  channel: 'predictions',
  filter: { station_id: 'station_001' }
}));

// 接收预测结果
ws.onmessage = (event) => {
  const data = JSON.parse(event.data);
  if (data.type === 'prediction') {
    console.log('New prediction:', data.payload);
    updateDashboard(data.payload);
  } else if (data.type === 'alarm') {
    console.log('Alarm triggered:', data.payload);
    showAlarmNotification(data.payload);
  }
};
```

## 🔄 数据更新策略

### 预测更新频率

| 预测类型 | 更新频率 | 触发方式 | 数据范围 |
|---------|---------|---------|---------|
| 超短期预测 | 5分钟 | 定时 + 事件 | 未来1小时 |
| 短期预测 | 1小时 | 定时 | 未来24小时 |
| 中期预测 | 6小时 | 定时 | 未来7天 |
| 长期预测 | 24小时 | 定时 | 未来30天 |

### 增量更新机制

```
1. 基准预测
   └─ 每天凌晨2:00生成
      └─ 覆盖未来30天

2. 增量更新
   ├─ 每小时更新
   │  └─ 修正未来24小时
   │
   └─ 事件触发
      ├─ 气象数据突变
      ├─ 设备状态变化
      └─ 实际值偏差过大
```

### 数据同步策略
