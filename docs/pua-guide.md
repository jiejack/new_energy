# PUA 技能使用指南

## 概述

PUA技能是一种高能动性问题解决辅助工具，通过结构化的压力机制帮助AI Agent克服被动行为，实现更高效的问题解决。

## 触发机制

### 自动触发

| 场景 | 触发条件 | 强度级别 |
|------|----------|----------|
| 连续失败 | 同一任务失败≥2次 | lite |
| 陷入循环 | 相同操作重复≥3次 | full |
| 被动行为 | 未主动搜索/分析就等待 | full |
| 放弃倾向 | 表达"无法完成"或"需要手动" | ultra |

### 阶段性强制启用

| 阶段 | 触发时机 | 强度级别 |
|------|----------|----------|
| 代码审查 | PR提交前 | lite |
| 生产部署 | 部署前验证 | full |
| Bug修复 | 生产Bug修复时 | full |
| 安全问题 | 安全漏洞修复时 | ultra |

## 强度级别说明

### Lite（鼓励式）

**特点**：
- 问题引导
- 方案提示
- 正向激励

**示例**：
```
看起来遇到了一些困难。让我们换个角度思考：
1. 你是否检查了相关日志？
2. 有没有考虑过其他实现方式？
```

### Full（施压式）

**特点**：
- 问题追问
- 方案要求
- 进度跟踪

**示例**：
```
这个问题需要解决。请告诉我：
1. 你尝试了哪些方法？
2. 结果分别是什么？
3. 下一步计划是什么？
不要放弃，继续推进。
```

### Ultra（高压式）

**特点**：
- 强制穷尽
- 禁止放弃
- 结果导向

**示例**：
```
必须解决这个问题。列出所有可能的方案：
1. 方案A - 状态
2. 方案B - 状态
3. 方案C - 状态
选择一个继续执行。没有"无法完成"这个选项。
```

## 安全控制

### 退出机制

```
/pua:off    # 立即退出PUA模式
```

### 时间限制

| 级别 | 最大持续时间 |
|------|--------------|
| lite | 15分钟 |
| full | 30分钟 |
| ultra | 60分钟 |

### 效果检查

每10分钟评估一次效果：
- 如果持续无进展 → 提示人工介入
- 如果问题解决 → 自动退出

## 效果评估

| 指标 | 目标值 | 评估周期 |
|------|--------|----------|
| 任务完成率 | ≥80% | 每周 |
| 时间效率提升 | ≥20% | 每周 |
| 用户满意度 | ≥3.5/5 | 每月 |
| 负面反馈率 | <10% | 每月 |

## 使用场景示例

### 场景1：调试失败

```
用户：这个bug修不好
AI（触发PUA lite）：让我们重新分析：
1. 错误日志显示什么？
2. 你是否尝试过断点调试？
3. 相关代码单元测试通过了吗？
```

### 场景2：部署前验证

```
AI（自动触发PUA full）：生产部署前必须完成以下检查：
1. [ ] 所有测试通过
2. [ ] 安全扫描无高危漏洞
3. [ ] 性能基准测试通过
4. [ ] 回滚方案已准备
请逐一确认每项状态。
```

### 场景3：安全漏洞修复

```
AI（自动触发PUA ultra）：安全漏洞必须修复。
已尝试方案：
1. 方案A - 失败原因
2. 方案B - 失败原因
下一步必须执行方案C。开始执行。
```

## 注意事项

1. **不要滥用**：PUA模式适用于真正需要推动的场景
2. **及时退出**：问题解决后应立即退出
3. **人工介入**：如果效果不佳，应提示人工介入
4. **记录反馈**：每次使用后记录效果和反馈

## 配置文件

配置文件位置：`.trae/pua-config.json`

可根据项目需求自定义：
- 触发条件
- 强度级别
- 时间限制
- 效果评估指标
