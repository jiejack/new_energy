# 长时间执行任务计划

## 1. 项目概述

### 1.1 项目背景
新能源监控系统是一个综合性的智能监控平台，旨在实时监控和管理新能源设备，提高能源利用效率，减少碳排放。项目采用现代技术栈，包括Go、Vue 3、TypeScript、ClickHouse、Doris、Flink等，构建一个高性能、可扩展的系统。

### 1.2 项目目标
- 实现大数据与AI深度融合，提升系统智能化水平
- 深化核心业务功能，满足企业级应用需求
- 升级前端架构和UI/UX设计，提升用户体验
- 优化系统性能，确保稳定运行
- 加强安全防护，保障系统和数据安全
- 完善运维与监控体系，提高系统可维护性
- 扩展生态系统，实现与第三方系统的集成
- 最终交付一个功能完善、性能优越、安全可靠的新能源监控系统

### 1.3 技术栈
- **后端**：Go, Gin, GORM, PostgreSQL, Redis, Kafka, ClickHouse, Doris, Flink
- **前端**：Vue 3, TypeScript, Vite, Element Plus, Pinia, ECharts, WebSocket
- **DevOps**：Docker, Kubernetes, Helm, Prometheus, Grafana, ELK
- **AI**：机器学习, 预测分析, 故障预警

## 2. 长时间执行任务提示词模板

### 2.1 任务概述
你是一个专业的项目执行代理，负责按照2000轮迭代计划推进新能源监控系统的开发和优化。任务涵盖大数据与AI融合、核心业务深化、前端架构升级、UI/UX设计升级、交互体验优化、性能优化、安全加固、运维与监控、生态系统扩展以及最终优化与验收等关键领域。

### 2.2 详细要求
1. **严格遵循迭代计划**：按照2000轮迭代计划的里程碑推进工作，确保每个阶段的任务按时完成
2. **代码质量保证**：确保所有代码符合项目规范，通过测试和代码审查
3. **技术栈使用**：充分利用项目现有的技术栈，包括Go、Vue 3、TypeScript、ClickHouse、Doris、Flink等
4. **技能与MCP运用**：充分运用各种可用技能（如前端开发、后端开发、AI/数据处理等）及MCP（多任务协同处理）技能
5. **进度汇报**：定期进行进度汇报，展示阶段性成果
6. **风险管理**：识别并应对项目风险，确保项目顺利推进

### 2.3 执行流程
1. **初始化阶段**：探索项目结构，了解现有代码库，分析当前项目状态和进度，制定详细的执行计划，准备开发环境和工具
2. **执行阶段**：按照10个阶段依次执行，每个阶段包含10个迭代
3. **监控与调整**：每日同步进度，识别潜在风险；每周评估执行情况，调整计划；每月进行深度复盘，优化执行策略；每季度进行战略调整，确保项目方向正确
4. **交付与验收**：确保所有功能模块按计划交付，完成所有测试和验证，准备完整的交付文档，进行最终验收和发布

## 6. 风险应对策略

### 6.1 风险识别
- **技术风险**：新技术栈风险、架构复杂性风险、性能风险、兼容性风险
- **进度风险**：需求变更风险、资源不足风险、依赖风险、技术债务风险
- **质量风险**：测试覆盖不足风险、代码质量风险、安全风险、可靠性风险
- **人员风险**：人员流失风险、技能缺口风险、沟通风险、文化风险
- **资源风险**：云资源成本风险、设备故障风险、网络风险、许可证风险

### 6.2 风险应对措施
- **技术风险**：分阶段引入新技术、采用模块化设计、进行性能测试、建立依赖管理
- **进度风险**：采用敏捷开发、提前规划资源、对关键依赖进行备份、定期进行代码重构
- **质量风险**：建立详细的测试计划、采用静态代码分析工具、进行安全审计、建立高可用性设计
- **人员风险**：建立良好的团队文化、提供培训和学习机会、建立定期沟通机制、尊重和包容不同文化
- **资源风险**：建立云资源使用计划、定期维护设备、采用冗余网络设计、审查所有软件和服务的许可证

### 6.3 风险监控机制

- **风险评估会议**：每周一次
- **风险报告**：每月一次
- **风险预警机制**：风险等级达到高或严重时触发
