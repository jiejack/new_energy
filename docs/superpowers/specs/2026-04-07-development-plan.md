### 2.3 配置规范

```yaml
# .trae/caveman-config.yaml
caveman:
  enabled: true
  default_intensity: "full"  # lite/full/ultra
  auto_trigger:
    conditions:
      - token_usage_above: 50000
      - conversation_length_above: 20
      - user_command: "/caveman"
  intensity_levels:
    lite:
      max_response_lines: 10
      code_only: false
    full:
      max_response_lines: 5
      code_only: true
    ultra:
      max_response_lines: 3

## 四、优化方向3：基于Harness Engineering核心理念的系统性优化

### 4.1 CI/CD流程瓶颈分析与改进

**当前瓶颈识别：**

| 瓶颈点 | 问题描述 | 影响 | 改进方案 |
|--------|----------|------|----------|
| 测试执行慢 | 全量测试耗时过长 | 阻塞发布 | 智能测试选择 + 并行执行 |
| 覆盖率检查 | 无增量覆盖率要求 | 质量风险 | 增量覆盖率门禁 |
| 安全扫描滞后 | 仅在发布前扫描 | 延迟发现问题 | 提交时预扫描 |
| 部署验证不足 | 缺少自动化验证 | 生产风险 | 金丝雀 + 自动回滚 |

**改进后CI/CD流程：**

```

**告警机制配置：**

```yaml
# deploy/prometheus/rules/harness_rules.yml
groups:
  - name: harness_critical
    rules:
      - alert: HarnessValidationFailed
        expr: harness_validation_errors_total > 0
        for: 1m
        labels:
          severity: critical
        annotations:
          summary: "Harness验证失败"
          
      - alert: HighLatencyP95
        expr: histogram_quantile(0.95, rate(http_request_duration_seconds_bucket[5m])) > 0.2
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "P95延迟超过200ms"
```

## 五、优化方向4：PUA技能使用规范

### 5.1 适用场景与边界条件

**自动触发场景：**

| 场景 | 触发条件 | 强度级别 |
|------|----------|----------|
| 连续失败 | 同一任务失败≥2次 | lite |
| 陷入循环 | 相同操作重复≥3次 | full |
| 被动行为 | 未主动搜索/分析就等待 | full |
| 放弃倾向 | 表达"无法完成"或"需要手动" | ultra |

**阶段性强制启用：**

| 阶段 | 启用时机 | 强度级别 |
|------|----------|----------|
| 代码审查 | PR提交前 | lite |
| 部署验证 | 生产发布前 | full |
| Bug修复 | 生产Bug修复时 | full |
| 安全问题 | 安全漏洞修复时 | ultra |

### 5.2 使用规范

```yaml
# .trae/pua-config.yaml
pua:
  enabled: true
  
  auto_trigger:
    enabled: true
    failure_threshold: 2
    loop_threshold: 3
    passive_detection: true
    
  phase_enforcement:
    code_review: "lite"
    pre_deploy: "full"
    bug_fix: "full"
    security_fix: "ultra"
    
  intensity_levels:
    lite:
      style: "鼓励式"
      features:
        - 问题引导
        - 方案提示
    full:
      style: "施压式"
      features:
        - 问题追问
        - 方案要求
        - 进度跟踪
    ultra:
      style: "高压式"
      features:
        - 强制穷尽
        - 禁止放弃
        - 结果导向
```

### 5.3 风险控制措施

| 控制措施 | 描述 | 实施方式 |
