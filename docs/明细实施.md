### 1.2 后端API模块

#### 1.2.1 告警规则API

**接口设计**：

| 方法 | 路径 | 描述 |
|------|------|------|
| GET | /api/v1/alarm-rules | 获取规则列表 |
| POST | /api/v1/alarm-rules | 创建规则 |
| GET | /api/v1/alarm-rules/:id | 获取规则详情 |
| PUT | /api/v1/alarm-rules/:id | 更新规则 |
| DELETE | /api/v1/alarm-rules/:id | 删除规则 |
| PUT | /api/v1/alarm-rules/:id/enable | 启用规则 |
| PUT | /api/v1/alarm-rules/:id/disable | 禁用规则 |

**实现步骤**：
1. 创建实体 `internal/domain/entity/alarm_rule.go`
2. 创建仓储接口 `internal/domain/repository/alarm_rule_repository.go`
3. 实现仓储 `internal/infrastructure/persistence/alarm_rule_repository.go`
4. 创建服务 `internal/application/service/alarm_rule_service.go`
5. 创建处理器 `internal/api/handler/alarm_rule_handler.go`

**验收标准**：
- [ ] 所有接口返回正确响应
- [ ] 参数验证正确
- [ ] 错误处理正确
- [ ] 数据持久化正确

**责任人**：后端开发工程师A

### 5.3 数据库设计

```sql
-- 告警规则表
CREATE TABLE alarm_rules (
    id VARCHAR(36) PRIMARY KEY,
    name VARCHAR(100) NOT NULL UNIQUE,
    type VARCHAR(20) NOT NULL,
    level INT NOT NULL,
    condition TEXT NOT NULL,
    threshold DECIMAL(10,4),
    duration INT DEFAULT 0,
    point_id VARCHAR(36),
    device_id VARCHAR(36),
    station_id VARCHAR(36),
    notify_channels TEXT,
    notify_users TEXT,
    status INT DEFAULT 1,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 通知配置表
CREATE TABLE notification_configs (
    id VARCHAR(36) PRIMARY KEY,
    type VARCHAR(20) NOT NULL UNIQUE,
    name VARCHAR(50),
    config JSONB,
    enabled BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 操作日志表
CREATE TABLE operation_logs (
    id VARCHAR(36) PRIMARY KEY,
    user_id VARCHAR(36),
    username VARCHAR(50),
    method VARCHAR(10),
    path VARCHAR(200),
    action VARCHAR(50),
    resource VARCHAR(50),
    resource_id VARCHAR(36),
    request_ip VARCHAR(50),
    status INT,
    duration BIGINT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```
